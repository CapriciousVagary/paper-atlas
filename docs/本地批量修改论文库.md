# 本地批量修改论文库

## 1. 批量导入论文

主数据文件是 `app/imported-papers.json`。这是网站中目前由脚本批量导入、此前自动补全内容的 53 篇论文。

每篇论文是一个 JSON 对象，常用字段如下：

- `slug`：站内详情页地址标识；已有论文尽量不要修改，否则旧链接会失效。
- `title`、`titleZh`：英文和中文标题。
- `doi`、`sourceUrl`：DOI 和论文来源链接。
- `journal`、`published`：期刊与发表年月。
- `category`、`subcategory`：主分类。
- `classifications`：论文同时属于的全部大类/小类。
- `authorDetails`：作者角色、姓名、单位、别名和说明。
- `abstractZh`：中文摘要。
- `insight`：50 字以内的一句话创新点。
- `keywords`：精简标签，建议不超过 6 个。
- `figureCaption`：关键图说明。
- `addedAt`：该条目进入网站的时间，用 ISO 8601 格式，例如 `2026-07-17T16:30:00+08:00`。

修改时必须保持合法 JSON：字段和字符串使用英文双引号，条目之间保留逗号。批量修改完成后运行网站构建检查；通过并重新发布后才会更新线上内容。

注意：如果某篇批量导入论文后来在在线审核台中修改过，云端的 `paper_edits` 覆盖记录优先于本地 JSON，同名字段可能遮盖本地修改。遇到“本地已改但线上仍显示旧值”时，需要检查或清除该论文的云端覆盖记录；不要直接删除整张表。

## 2. 分类定义

预设大类、小类及其首页说明位于 `app/data.ts` 的 `categories` 数组。论文可使用自定义分类；但若要让某一分类成为首页固定预设，应在这里同步调整。

## 3. 线上投稿论文

访客从网站上传并经审批的论文保存在云端数据库中，不会写回 `app/imported-papers.json`。这类论文目前请在 `/admin` 的“编辑已收录论文”中修改。

如需在本地批量处理线上投稿论文，应先增加“导出为 JSON / 修改后重新导入”功能，不能直接在本地文件中找到它们。
